# 🚗 AutoRental System (Legacy Project)

> Був запит на переведення проєкту у публічний доступ, тому код викладено в GitHub у відкритому вигляді


> **⚠️ Увага:** Даний проект є одним з моїх перших фріланс-замовлень. Код написаний достатньо давно і на поточний момент вважається **Ultra Legacy**. Архітектура та підходи можуть не відповідати сучасним стандартам (особливо в частині безпеки), але проект повністю функціональний і демонструє мій шлях та навички роботи з повним стеком технологій на той період.

### 🌟 Огляд проекту
Система для автоматизації прокату автомобілів. Включає в себе:
- **Кабінет користувача**: пошук, детальна фільтрація та бронювання авто.
- **Система штрафів**: відстеження порушень та статусів оплати.
- **Панель адміністратора**: повне керування автопарком, користувачами (бан/ролі) та аналітика бронювань через дашборди.

### 🛠 Стек технологій
- **Backend:** .NET 8 (ASP.NET Core), Entity Framework Core, Clean Architecture.
- **Frontend:** React, Next.js, Tailwind CSS.
- **Storage:** PostgreSQL (DB), MinIO (S3-сумісне сховище для фото автомобілів).
- **DevOps:** Docker Compose.

## 🚀 Як запустити (Інструкція для розробника)

Щоб підняти проект, вам потрібно встановити:

- 🐳 [Docker](https://www.docker.com/get-started)
- 🟢 [Node.js (LTS)](https://nodejs.org/)

### 1. Підготовка бази та сервісів (Backend)
Backend частина та необхідна інфраструктура контейнеризовані.
1. Перейдіть в папку: `cd /AutoRentalSystem.API`
2. Запустіть сервіси: `docker-compose up -d`

   *Це підніме API, базу даних PostgreSQL, адмінку для бази та сховище MinIO.*

### 2. Налаштування даних (Database & MinIO)
Для коректної роботи потрібно імпортувати початкові дані:
- **База даних:** Використайте файл `.files/dump.sql` (додається в проекті) для відновлення структури та даних у вашій PostgreSQL.

#### 🐘 Ініціалізація бази даних
Виконайте команду для імпорту дампу в PostgreSQL:

```bash
docker exec -i auto_rental_postgres psql -U postgres -d AutoRental < .files/dump.sql
```

### 🚀 Інструкція з налаштування MinIO 

#### 1. Встановлення MinIO Client (mc)
Виберіть спосіб встановлення залежно від вашої операційної системи:

#### **Windows (через Scoop)**
Якщо у вас встановлено менеджер пакетів Scoop, виконайте:
```bash
> `scoop install mc`
```
#### **Linux / WSL**
Для систем на базі Linux або Windows Subsystem for Linux використовуйте наступні команди:
1. Завантажте бінарний файл:
   `curl -O https://dl.min.io/client/mc/release/linux-amd64/mc`
2. Надайте права на виконання:
   `chmod +x mc`

---

### 2.2 Підключення до MinIO
Створіть псевдонім для вашого локального сервера, щоб зручно звертатися до нього:
```bash
`mc alias set local http://localhost:9000 admin secret123`
```
*Параметри: назва (`local`), URL (`http://localhost:9000`), логін (`admin`) та пароль (`secret123`).*

---

#### 2.3 Створення бакету
Створіть новий контейнер з назвою `cars-images`:

```bash
`mc mb local/cars-images`
```
---

#### 2.4 Розпакування архіву із зображеннями
Фото знаходяться в архіві `images.zip`, розпакуйте їх у папку `images`:
```bash
mc cp images.zip local/cars-images/
```
---

#### 2.5 Завантаження зображень у MinIO
Завантажте вміст локальної папки у створений бакет на сервері:
```bash
`mc cp --recursive ./images local/cars-images`
```
---

#### 2.6 Перевірка завантажених файлів
Переконайтеся, що всі файли успішно завантажені, переглянувши вміст бакету:
```bash
`mc ls local/cars-images`
```
---

### 3. Запуск фронтенду
Фронтенд запускається локально через менеджер пакетів npm:
1. Перейдіть в папку: `cd /AutoRentalSystem.Frontend/autorentalsystem`
2. Встановіть залежності: 
```bash 
npm install 
```
3. Запустіть проект: 
```bash 
npm run dev 
```
4. Відкрийте у браузері: `http://localhost:3000`
